bad

  1. 用一个函数包裹另一个函数，为了延迟执行

    • improve

    • why

      • 徒增代码量
      • 不好维护
      • 如果内部函数有变动，外部的函数也需要有改动

  2. 变量命名不清晰，如果是泛用的函数，不要定义为特定的场景命名

    • improve

    • why

      • 容易限定数据，重复代码

  3. 提防this

  • improve
  • why
    • this的指向很难控制，有时候考虑不全
    • 改进的方式是将this绑定到Db对象上

how to improve

  1. pure function

    1. 什么是纯函数：相同的输入会得到相同的输出，并且无副作用
    2. 副作用
      • 引起数据的改变
      • 依赖外部数据，随着外部数据的变化而变化
      • 依赖系统状态
    3. 为什么用纯函数
      • 可缓存
      • 可移植和自成说明
      • 可测试
      • 合理
      • 并行代码

  2. curry

curry：只传递一部分参数给函数调用，并返回一个函数处理余下的参数

example

只传一部分参数，局部调用 partial application
